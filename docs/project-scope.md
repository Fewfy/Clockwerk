# Clockwerk 项目范围说明

## 目标

Clockwerk 聚焦机器狗系统中的高层决策闭环。

这一阶段只解决一个明确问题：在有限的结构化观测下，稳定地决定当前应处于哪种高层状态，以及应向低层控制层发出什么类型的动作指令。

项目当前不追求通用机器人平台，不覆盖完整感知栈，也不试图一次性整合视觉、语音、地图或大模型能力。

## 范围边界

当前版本只包含四个高层状态：

- `IDLE`
- `NAVIGATE`
- `RECOVER`
- `EMERGENCY_STOP`

边界约束如下：

- 高层决策只负责状态选择和动作意图分发
- 连续控制、步态细节和电机级控制不在本层实现
- 安全保护独立存在，并且优先级高于普通决策
- 先支持规则化决策闭环，再考虑更复杂的策略替换

## 最小输入

为了保证系统可解释、可测试、可扩展，第一阶段只接收以下输入：

- 目标方向或目标点
- IMU 姿态
- 机体速度
- 障碍物距离
- 跌倒状态

这些输入足以支撑四种高层状态之间的第一版切换逻辑。

以下内容暂不纳入最小输入集合：

- 原始视觉图像
- 语音指令
- 全局语义地图
- 高维传感器特征

## 最小输出

系统第一阶段只产出两类结果：

- 高层状态切换结果
- 低层运动命令或技能命令

输出约束如下：

- 高层状态用于描述系统当前意图
- 低层命令用于驱动移动、停止或恢复等基础能力
- 不在高层直接输出电机级控制量

## 非目标

当前阶段明确不做以下事项：

- 通用具身智能平台
- 多机器人协同
- 端到端学习控制
- 完整 SLAM 或大规模地图构建
- 复杂多模态人机交互

## 第一阶段验收标准

如果满足以下条件，就认为项目边界已经被正确定义：

- 所有人对四个高层状态的职责有一致理解
- 新增模块时可以判断它是否属于当前范围
- 后续架构、接口和状态机设计都以这组最小输入输出为基础
